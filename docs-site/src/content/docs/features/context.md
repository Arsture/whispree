Whispree는 “무엇을 보며 말하는가”와 “어디에 넣어야 하는가”까지 신경 씁니다. 화면 컨텍스트는 비전 모델 교정에, 컨텍스트 복원은 정확한 삽입 위치에 쓰입니다.

## 화면 컨텍스트 (비전)

비전 가능한 교정 모델을 쓸 때, 녹음 중 화면을 캡처해 모델에 함께 전달하면 화면 속 용어·코드·이름을 반영한 더 정확한 교정을 받을 수 있습니다.

설정 → LLM(비전 모델 선택 시 표시):

| 설정 | 하는 일 | 기본 |
| --- | --- | --- |
| **스크린샷 컨텍스트 활성화** | 녹음 중 화면을 캡처(앱 전환/스크롤/클릭 디바운스, 최대 20장)해 비전 모델에 전달. **화면 기록 권한 필요.** | 꺼짐 |
| **에이전트에 전달** | 텍스트 삽입 후, 캡처한 스크린샷을 이미지로도 대상 앱에 붙여넣기. | 꺼짐 |

**캡처는 지능형 디바운스**로 동작합니다 — 앱을 바꾸거나 스크롤/클릭하면 타이머가 리셋되고, 1.5초 멈추면 캡처합니다(최대 20장, 메모리 보호).

### 스크린샷 선택 패널
변환이 끝나면 어떤 스크린샷을 붙일지 고르는 패널이 뜹니다(여러 녹음이 큐에 있으면 FIFO 순서대로):

- **↑ ↓** 이동 · **Space** 선택 · **Enter** 확정 · **⌘Enter** 미리보기(퀵룩 스타일) · **Esc** 건너뛰기.

### 녹음 중 즉석 토글 (⌥)
녹음 도중 **왼쪽 Option 키를 ~0.5초 길게** 누르면 “스크린샷을 대상 앱에 첨부” 를 켜고 끌 수 있습니다. 오버레이에 `Img Attach ON/OFF` 가 잠깐 표시됩니다(비전 모델이 활성일 때만).

## 컨텍스트 복원

받아쓰기를 시작할 때 쓰던 자리를, 삽입 시점에 정확히 되돌립니다. 단순히 “현재 맨 앞 앱”에 붙이는 게 아니라 **녹음 시작 시점의 탭/입력란**으로 돌아갑니다.

### 브라우저 복원 (Chrome)
설정 → 일반의 **브라우저 복원** *(기본 켜짐)*. 녹음 시작 시의 **Chrome 탭과 포커스된 입력 요소**를 기억했다가, 삽입 전에 그 탭·그 입력란으로 돌아갑니다. 처리 중 같은 입력란에서 더 입력하거나 커서를 옮긴 뒤 다른 앱으로 이동했다면, 삽입 직전에 Chrome의 최신 커서 위치를 다시 읽어 그 자리에 붙여넣습니다.

- Chrome 전용. 자동화(Automation) 권한이 필요합니다.
- 요소 단위 커서 복원에는 Chrome의 “Apple Events에서 JavaScript 허용”이 필요합니다. 꺼져 있어도 **탭 복원은 동작**합니다(요소 복원만 생략).

### 터미널 복원 (iTerm2 / tmux)
설정 → 일반의 **터미널 복원** *(기본 켜짐)*. **iTerm2 세션(분할 pane)** 과, 그 안에서 tmux가 붙어 있으면 **tmux 윈도우/페인**까지 기억했다가 그 자리로 복원합니다.

- iTerm2 전용. 자동화 권한이 필요합니다. tmux는 기본 소켓만 지원합니다.
- Terminal.app · Alacritty · Kitty · Ghostty · Warp는 분할/세션 복원을 지원하지 않습니다(앱 포커스 복원만).

:::note
컨텍스트 복원은 best-effort입니다. 복원에 실패해도 받아쓰기 결과 삽입 자체는 진행됩니다.
:::

필요한 권한 설정은 [권한](/reference/permissions/)을 참고하세요.
