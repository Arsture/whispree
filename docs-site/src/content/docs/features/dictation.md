받아쓰기는 Whispree의 핵심 흐름입니다. **녹음 → STT → (선택) AI 교정 → 직전 앱에 삽입**이 단축키 한 번으로 이어집니다.

## 기본 흐름

1. 단축키(`Ctrl+Shift+R`)로 녹음 시작.
2. 말하는 동안 파형 오버레이가 상태와 단축키 배지를 표시.
3. 녹음을 멈추면 STT가 변환하고, 교정 엔진이 켜져 있으면 다듬은 뒤,
4. **녹음을 시작하기 직전에 쓰던 앱**의 입력란으로 결과가 삽입됩니다(클립보드 + 붙여넣기).

대상 앱을 따로 고를 필요는 없습니다. Whispree는 녹음 시작 시점의 앱을 기억해 두었다가 그 자리로 되돌아갑니다. 유효한 대상이 없으면(예: 설정 창에서 녹음) 클립보드 복사로 대체합니다.

## 녹음 모드

설정 → 일반에서 선택합니다.

| 모드 | 동작 | 추천 상황 |
| --- | --- | --- |
| **눌러서 말하기(Push-to-Talk)** *(기본)* | 단축키를 누르고 있는 동안 녹음, 떼면 변환 | 짧게 자주 말할 때 |
| **토글(Toggle)** | 한 번 눌러 시작, 다시 눌러 멈춤 | 길게 이어 말할 때 |

## 멈추지 않는 멀티 녹음

Whispree는 녹음과 후처리를 분리합니다. 그래서 **이전 발화가 변환·교정되는 동안 바로 다음 녹음을 시작**할 수 있습니다.

- 녹음을 멈추면 그 발화는 즉시 **작업(job)** 으로 큐에 들어갑니다(설정·오디오·대상·스크린샷 스냅샷 포함).
- STT와 AI 교정은 **프로바이더별 동시 실행 한도** 안에서 병렬 처리됩니다.
- 최종 **삽입은 항상 말한 순서대로(FIFO)** 직렬 처리됩니다 — 나중 작업이 먼저 끝나도 순서를 지킵니다.
- 삽입은 **녹음 중에는 멈춰** 있다가, 녹음이 끝나면 큐의 맨 앞부터 이어집니다.

오버레이/대시보드는 조용한 카운트(대기 N · 처리 N)로 상태를 보여줍니다.

:::tip
받아쓰기 결과를 기다리며 멍하니 있을 필요가 없습니다. 생각나는 대로 계속 말하고, Whispree가 순서대로 정리해 넣어줍니다.
:::

## ESC — 범위만 취소

ESC는 **현재 보이는 작업 하나만** 취소합니다. 배경에서 처리 중인 전체 큐를 한꺼번에 지우지 않습니다. 취소 범위는 상황에 따라: 미리보기 → 화면 선택/삽입 중 → 진행 중인 녹음 → 화면에 표시된 대기 항목(`Cancel #N`) 순입니다.

## 외부에서 제어 (URL 스킴)

Raycast, Stream Deck, Keyboard Maestro, AppleScript 등에서 URL로 녹음을 제어할 수 있습니다.

| URL | 동작 |
| --- | --- |
| `whispree://toggle` | 녹음 토글 |
| `whispree://start` (또는 `whispree://push`) | 녹음 시작 |
| `whispree://stop` (또는 `whispree://release`) | 녹음 종료 |

## 메뉴바 & 오버레이

- **메뉴바 아이콘** — 녹음/대기 상태를 표시하고, 클릭하면 메인 윈도우가 열립니다.
- **파형 오버레이** — 녹음/변환/교정 단계, 파형, 그리고 Stop · `Cancel esc` · `Img Attach ⌥` 같은 단축키 배지를 보여줍니다. 설정 → 일반의 **오버레이 표시** 로 끌 수 있습니다.

전체 단축키 목록은 [단축키](/reference/shortcuts/)를, 내부 큐 구조는 [구조 한눈에](/reference/architecture/)를 참고하세요.
