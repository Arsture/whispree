---
title: 권한
description: Whispree가 요청하는 macOS 권한과 각 권한이 어떤 기능에 쓰이는지.
---

Whispree는 음성을 녹음하고, 화면을 보고, 다른 앱에 텍스트를 넣기 때문에 몇 가지 macOS 개인정보 보호 권한이 필요합니다. **마이크와 손쉬운 사용만 있으면 기본 받아쓰기는 완전히 동작**하고, 나머지는 해당 기능을 쓸 때만 필요합니다.

## 권한 한눈에

| 권한 | 쓰이는 곳 | 없으면 나타나는 증상 |
| --- | --- | --- |
| **마이크** | 음성 녹음(STT) | 녹음에 소리가 잡히지 않음 |
| **손쉬운 사용(Accessibility)** | 결과를 직전 앱에 붙여넣기(키 이벤트 전송) | 클립보드 복사만 되고 자동 삽입이 조용히 실패 |
| **화면 기록(Screen Recording)** | [화면 컨텍스트](/features/context/)용 스크린샷 캡처 | 비전 교정에 스크린샷이 들어가지 않음 |
| **자동화(Automation)** | [브라우저/터미널 복원](/features/context/) (AppleScript) | Chrome/iTerm2 컨텍스트 캡처·복원만 실패 |
| **앱 관리(App Management)** | 자동 업데이트(선택) | 자동 업데이트 적용 제한 |

## 부여 방법

대부분 해당 기능을 **처음 쓸 때 macOS가 자동으로 프롬프트**를 띄웁니다. 수동으로 켜려면 **시스템 설정 → 개인정보 보호 및 보안**에서 항목별로 Whispree를 허용하세요. 앱의 설정 → 일반(과 첫 실행 온보딩)에도 권한 상태 패널과 시스템 설정 바로가기가 있습니다.

자동화는 **대상 앱별로** 따로 허용됩니다 — 예: Apple Music, Spotify, Google Chrome, iTerm2. 각 기능을 처음 쓸 때 요청됩니다.

:::caution
**자동화 함정**: 브라우저/터미널 복원이 조용히 안 될 때가 있습니다. AppleScript 권한 프롬프트는 권한이 한 번도 부여되지 않았으면 표시되지 않고 실패만 남을 수 있습니다. 시스템 설정 → 개인정보 보호 및 보안 → 자동화에서 Whispree 아래 대상 앱(Chrome/iTerm 등)이 켜져 있는지 확인하세요.
:::

## 권한과 기능 매핑

- 기본 받아쓰기·삽입 → **마이크 + 손쉬운 사용**
- 화면 컨텍스트(비전 교정) → **화면 기록**
- 브라우저/터미널 복원 → **자동화**(대상 앱별)
- 녹음 중 음악 일시정지 → 음악 앱 제어 시 **자동화**(Apple Music/Spotify)

기능별 자세한 동작은 각 [기능](/features/dictation/) 문서를 참고하세요.
